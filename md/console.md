# Консоль и вывод на экран (`src/console.rs`, `src/font.rs`)

Документация к текстовому выводу Tomorrow OS. Графического стека нет: ядро
рисует **прямо в линейный framebuffer**, полученный от GRUB, попиксельно
выкладывая глифы шрифта. Терминал — это по сути «пишущая машинка»: курсор
`(cx, cy)` в клетках символов, при переполнении экран **очищается целиком**
(скролла нет). Один общий `Console` используется и для boot-логов, и для шелла.

> Framebuffer — 32 бита на пиксель (`u32`, `0x00RRGGBB`). Текст белый
> (`0xFFFFFF`) на чёрном (`0x000000`). Шрифт — **PSF2**, моноширинный 8×16,
> вшит в бинарь через `include_bytes!("font.psf")`.

---

## 1. Откуда берётся framebuffer

Адрес и геометрия приходят из multiboot2-тега framebuffer (разбирается в
`main.rs`) и складываются в структуру `Console`:

```rust
pub struct Console {
    pub fb: *mut u32,   // база линейного framebuffer (identity-mapped)
    pub pitch: u32,     // байт на строку пикселей (может быть > width*4)
    pub width: u32,     // ширина в пикселях
    pub height: u32,    // высота в пикселях
    pub cx: u32,        // курсор: колонка в СИМВОЛАХ (не пикселях)
    pub cy: u32,        // курсор: строка в символах
}
unsafe impl Send for Console {}
```

Ключевые тонкости адресации:

- **`pitch` ≠ `width * 4`.** Строки framebuffer могут быть выровнены с запасом,
  поэтому индекс пикселя считается через `stride = pitch / 4`:
  `fb[py * stride + px]`. Использовать `width` вместо `stride` — классический
  баг «картинка едет по диагонали».
- **`cx`/`cy` — в клетках символов**, пиксельные координаты получаются
  умножением на `GLYPH_W`/`GLYPH_H`.
- Глобальный экземпляр живёт в `static mut CONSOLE: Option<Console>`; доступ —
  через хелпер (`(&raw mut CONSOLE).as_mut()...`), тот же паттерн, что у шелла.

## 2. Рендер глифа (`font.rs`)

Шрифт — **PSF2**. Заголовок (32 байта): magic, version, `headersize`, flags,
numglyph, `bytesperglyph`, height, width. Для нашего 8×16 — `bytesperglyph = 16`
(по одному байту на строку глифа, 8 бит = 8 пикселей ширины).

```
glyph(c):  offset = headersize + c * bytesperglyph   → срез в 16 байт
draw_char: для каждой из 16 строк берём байт;
           бит (0x80 >> col) выставлен → рисуем пиксель fg
```

`draw_char(c, x, y, fg, fb, pitch)` кладёт один символ в пиксельных координатах
`(x, y)`. Прозрачность: рисуются **только** установленные биты — фон не
затирается. Отсюда важное следствие для стирания (см. §4).

| Константа | Значение |
|-----------|----------|
| `GLYPH_W` | `8` (пикселей) |
| `GLYPH_H` | `16` (пикселей) |

## 3. API вывода (`Console`)

| Метод | Что делает |
|-------|-----------|
| `write_byte(b)` | печатает один байт; `'\n'` → новая строка; при переполнении `cx` — авто-перенос |
| `write_str(s)` | побайтовый вывод строки |
| `write_hex(v)` | 16 hex-цифр (u64), фиксированная ширина |
| `write_dec(v)` | десятичное число (u64) через временный буфер |
| `backspace()` | сдвигает курсор назад и **заливает клетку фоном** |
| `clear()` | заливает весь экран чёрным, `cx = cy = 0` |

Продвижение курсора в `write_byte`:

```
'\n'           → cx = 0; cy += 1
обычный символ → draw_char(...); cx += 1
                 если cx*GLYPH_W >= width → cx = 0; cy += 1   (перенос строки)
после всего    → если cy >= height/GLYPH_H → clear()          (переполнение экрана)
```

---

## 4. Грабли

1. **Скролла нет — только `clear()`.** Когда курсор уходит ниже последней строки
   (`cy >= height/GLYPH_H`), экран очищается целиком, вывод начинается сверху.
   Это осознанное упрощение; полноценный скролл потребовал бы сдвига
   framebuffer вверх на строку глифов.

2. **Пробел — пустой глиф, им нельзя стирать.** В `draw_char` рисуются только
   установленные биты, а у `' '` их нет — «печать пробела» ничего не сотрёт
   поверх старого символа. Поэтому `backspace()` **вручную заливает всю клетку**
   `GLYPH_W × GLYPH_H` фоновым цветом, а не печатает пробел.

3. **`pitch`, а не `width`, при индексации.** Всегда `stride = pitch / 4`.
   Путаница даёт «диагональную» картинку на дисплеях, где строка длиннее ширины.

4. **`unsafe impl Send`.** Раздаётся вручную, потому что `*mut u32` не `Send` по
   умолчанию. Безопасно, только пока доступ к консоли не конкурентный — а он
   сейчас сериализуется через отключение прерываний в критических путях (см.
   line discipline в `shell.md`).

## 5. Карта функций

| Файл | Функция | Роль |
|------|---------|------|
| `console.rs` | `write_byte/str/hex/dec` | вывод текста и чисел |
