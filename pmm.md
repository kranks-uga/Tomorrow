# Фаза 6: PMM (Physical Memory Manager)

## Что это и зачем

PMM отслеживает какие физические страницы памяти свободны, а какие заняты.
Страница = 4KB = 4096 байт.

Без PMM нельзя сделать VMM, heap, процессы — всё что выделяет память.

## Как работает bitmap PMM

Храним массив битов: 1 бит = 1 страница.
- бит `0` → страница свободна
- бит `1` → страница занята

Для 16GB RAM нужно: `16 * 1024 * 1024 * 1024 / 4096 / 8 = 524288` байт = 512KB под bitmap.

## Где взять карту памяти

Multiboot2 передаёт memory map — тег типа `6`.
Каждая запись содержит:
- `base_addr: u64` — начало региона
- `length: u64` — размер
- `typ: u32` — тип (1 = свободна, всё остальное = занято)

## Что реализовать

### Структуры
```
MbMemoryMap      — тег типа 6 из multiboot2
MbMemoryEntry    — одна запись карты памяти
```

### Функции в src/pmm.rs
```
init(mmap_addr, mmap_size, entry_size)
    — читает записи, помечает свободные регионы как free
    — важно: регион где живёт сам kernel помечаем как занятый!

alloc() -> u64
    — ищет первый свободный бит, помечает как занятый, возвращает физ. адрес

free(addr: u64)
    — помечает страницу как свободную
```

### Глобальный bitmap
```rust
static mut BITMAP: [u64; N] = [0; N];  // все заняты по умолчанию
static mut TOTAL_PAGES: u64 = 0;
```

## Порядок работы

1. Парсим memory map из multiboot2 (тег 6) — сохраняем адрес
2. Вызываем `pmm::init(...)` в kernel_main
3. Помечаем свободные регионы
4. Помечаем регион ядра как занятый (адрес ядра знаем из linker.ld)
5. Проверяем: `pmm::alloc()` должен вернуть ненулевой адрес

## Адрес ядра

В `linker.ld` есть символы `_kernel_start` и `_kernel_end`.
В Rust их можно получить через:
```rust
extern "C" {
    static _kernel_start: u8;
    static _kernel_end: u8;
}
```
